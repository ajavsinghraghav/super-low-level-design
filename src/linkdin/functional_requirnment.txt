# LinkedIn  system low-level design
Functional Requirements
1-System should allow the user to create an account with their professional information, such as name, email, and password.
2- The user should be able to login and logout from his account
3-Each user should have a profile with his important user professional information  such as a profile picture, headline , summary,education
 skills and experience, Location,
4 users should be able to send connection requests to another user
5- The user should be able to accept or decline a connection request
6- user should be able to view their connection list;
7-user should be able to send a message to their connection
9-user should be able to view his inbox and send messages
8- Employee should be able to post a job listing with details such as title, description , requirements,and location
9- The system should allow the user to view and apply for job posting
10-system should allow the user to search for other users' companies and job posting

11- search results should be  ranked based on relevance and user preference
12-user should receive notification for events such as connection requests, messages, and job postings,
13-Notification should be delivered in real-time
14-The System should be designed to handle a large number of concurrent users and high-traffic load
15-System should be scalable and efficient in terms of resource utilization
